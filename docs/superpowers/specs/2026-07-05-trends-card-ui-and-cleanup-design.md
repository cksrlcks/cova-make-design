# 트렌드 레퍼런스 + 카드형 인터뷰 + 설치기/토큰 정리 — 설계

날짜: 2026-07-05
상태: 승인됨 (정적 트렌드 파일 방식 채택)

## 배경

현재 스킬 결과물이 "동일한 이너폭(.inner) 안에 갇힌 전형적인 형태"로 수렴해 시안 간 차별점이
없다. 원인은 (1) 참고 데이터가 COVA 패턴뿐이고 (2) design-rules.md의 "모든 콘텐츠는 .inner
안에" 규칙이 획일화를 강제하기 때문. 아울러 인증이 웹 로그인 플로우로 넘어갔으므로 수동 토큰
(`COVA_DESIGN_TOKEN`) 경로와 설치기의 관련 안내는 제거하고, 설치기 UX(방향키 선택)와
인터뷰 UX(카드형 질문)를 개선한다.

## 요구사항 (5개)

### R1. 설치 완료 메시지 정리

[bin/install.mjs](../../../bin/install.mjs)의 "다음 단계" 안내에서 스킬 동작 방법 한 줄만 남긴다:

- 유지: `Claude Code에서 "AI 시안 만들기"라고 요청하면 스킬이 동작합니다.`
- 삭제: API 주소 기본값 안내 줄, 업로드 인증 안내 줄.

### R2. `COVA_DESIGN_TOKEN` 기능 삭제

인증은 웹 로그인 플로우 하나로 통일한다. 로그인으로 발급된 토큰을 `~/.cova/credentials`에
저장·재사용하는 동작은 유지한다.

- `SKILL.md` 기본 설정: `TOKEN="${COVA_DESIGN_TOKEN:-...}"` → 저장된 자격증명만 읽음
  (`TOKEN="$(cat ~/.cova/credentials 2>/dev/null)"`), 주석의 우선순위 설명도 갱신.
- `README.md` 환경변수 표에서 `COVA_DESIGN_TOKEN` 행 삭제 (`COVA_API_URL`만 남음).
- `install.mjs`에는 R1 적용 후 토큰 언급이 없으므로 추가 작업 없음.
- 6단계 웹 로그인 플로우(세션 생성 → 폴링 → 저장)는 변경하지 않는다.

### R3. 인터뷰 질문을 항상 카드형식(AskUserQuestion)으로

- `SKILL.md` 1단계와 `references/interview.md`에 명시: 사용자에게 묻는 모든 인터뷰 질문은
  **AskUserQuestion 도구(카드 UI)** 로 묻는다. 일반 텍스트로 질문을 나열하지 않는다.
- interview.md의 질문들을 카드에 맞게 재구성:
  - 선택지형(페이지 목적, 페이지 타입, 대표 CTA, 이미지 톤, 태그 선택 흐름, 업로드 여부,
    섹션 구성[multiSelect]) → 옵션 카드로 제시. 각 질문에 넣을 옵션 예시를 문서에 적어 둔다.
  - 자유 입력형(사이트 제목, 업종, 슬로건, 추가 요청사항 등) → 카드의 "Other/직접 입력"을
    활용하되, 대표적인 예시 옵션을 2~3개 같이 제시해 선택도 가능하게 한다.
  - 한 호출에 최대 4개 질문까지 묶을 수 있으므로 "필수 → 홈페이지용 → 마무리" 묶음 단위로
    2~3회에 나눠 묻는다(기존 단계적 흐름 유지).

### R4. 웹디자인 트렌드 레퍼런스 추가 (정적 파일)

새 파일 `skills/cova-make-design/references/design-trends.md`를 만든다. 내용은 구현 시점에
웹 리서치로 수집해 정리하며, 스킬에 정적으로 포함된다(실행 시 웹검색 없음 — 갱신은 리서치를
다시 돌려 파일만 업데이트).

**수집원:**
- 국내: GDWEB(지디웹), DBCUT(디비컷) 수상작/셀렉션 경향
- 해외: Awwwards(SOTD/연간 수상작), 디자인 블로그·아티클(트렌드 리포트류)
- 기간: 최근 3년(2023~2026)

**문서 구조 (생성에 바로 쓰이는 형태):**
1. 연도별 흐름 요약(2023→2026) — 방향성 파악용, 짧게.
2. 국내 경향 vs 해외 경향 — 톤·밀도·타이포·레이아웃 차이. 업종별 국내 관행 포함.
3. **디자인 무브 카탈로그(핵심)** — 트렌드를 실행 가능한 레이아웃/타이포/모션 지시로 번역:
   풀블리드 히어로, 비대칭 그리드, 오버사이즈 타이포, 요소 겹침(오버랩), 이너박스 이탈
   섹션, 가로 스크롤 갤러리, 빅 풋터, 그리드 노출형 레이아웃 등. 각 무브에 "언제 쓰나 /
   어떻게 구현하나(CSS 힌트) / 과용 금지 조건"을 붙인다.
4. 안티 트렌드(이미 식상해진 것) — 보라-파랑 그라데이션 등 기존 금지 목록과 통합적 관점.

**연결 작업:**
- `SKILL.md` 4단계(HTML 생성)에서 design-rules.md와 함께 design-trends.md도 **반드시 읽도록**
  명시하고, 시안마다 트렌드 무브를 최소 1~2개 의도적으로 채택하게 한다.
- `design-rules.md`의 ".inner 강제" 규칙 완화: 좌측 기준선 원칙은 유지하되, 페이지당 1~2개
  섹션은 의도적으로 풀폭/비대칭/오버랩으로 리듬을 깨도록 허용·권장. "섹션 다양성" 절과
  트렌드 파일을 상호 참조.
- `README.md` 구성 트리에 design-trends.md 추가.

### R5. 설치 위치 방향키 선택

`install.mjs`의 숫자 입력 프롬프트를 ↑/↓ + Enter 인터랙티브 선택으로 교체한다.

- **의존성 없이** Node 내장 `readline`(raw mode) + ANSI 이스케이프로 직접 구현
  (npx 설치기라 외부 패키지는 설치 지연만 유발).
- 동작: 두 옵션(전역/현재 프로젝트)을 세로로 표시, 선택 행에 `❯` 포인터+강조.
  ↑/↓(및 j/k)로 이동, Enter로 확정, Ctrl+C로 취소(exit 1). 확정 후 선택 결과를 한 줄로 남김.
- 숫자 입력(1/2)도 계속 허용해 하위 호환.
- 비TTY/CI 폴백(질문 없이 전역 + 안내 문구)과 `--global`/`--project`/`--help` 플래그는
  현행 유지.

## 접근 방식 결정

- 트렌드 데이터: **정적 파일**(승인됨). 실행 시 웹검색 보강안은 채택하지 않음 — 실행 속도와
  결과 일관성 우선.
- 방향키 프롬프트: **제로 의존성 직접 구현**. 라이브러리(@clack/prompts 등) 대비 코드는 늘지만
  npx 첫 실행 속도와 공급망 단순성이 이득.

## 테스트

- `install.mjs`: TTY에서 방향키/j·k/숫자/Enter/Ctrl+C 동작 수동 확인, `--global`·`--project`·
  비TTY(`echo | node bin/install.mjs`) 회귀 확인, 설치 후 dest에 design-trends.md 포함 확인.
- `SKILL.md`/references: 스킬을 실제 발동해 (1) 질문이 카드로 나오는지 (2) 생성물이 트렌드
  무브를 반영해 이너박스 획일화를 벗어나는지 (3) `COVA_DESIGN_TOKEN` 없이 로그인 플로우만으로
  업로드되는지 확인.
- grep으로 `COVA_DESIGN_TOKEN` 잔존 참조 0건 확인(docs/ 이력 문서는 제외).

## 비범위

- 웹 로그인 플로우(6단계) 로직 변경 없음.
- COVA API·서버 측 변경 없음.
- 기존 스펙/플랜 문서(docs/)는 이력이므로 수정하지 않음.
