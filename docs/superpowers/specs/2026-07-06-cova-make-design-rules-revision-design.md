# cova-make-design 규칙 4종 개정 — 설계

날짜: 2026-07-06
상태: 승인됨 (트렌드 목적 재정의 / 페이지 유형 게이트 승격 / .inner 의무 폐기 / 과감함·풀스크린 히어로 기본 상향)

## 배경

`cova-make-design`은 홈페이지 시안 HTML 한 장을 생성하는 스킬이다. 운영 경험을 반영해 디자인 규칙과
인터뷰 흐름을 다음 4가지로 개정한다.

1. **트렌드 사용 목적 재정의** — 트렌드는 "전형성 회피"를 넘어 **다양한 베리에이션·무드·색감·레이아웃을
   차용**하기 위한 참고 자료로 재규정한다.
2. **페이지 유형 게이트 승격** — 시안을 시작할 때 어떤 페이지(메인/서브/관리자)를 만들지 사용자가 말하지
   않았으면 **먼저 묻는 게이트**로 올린다. 이 선택은 COVA studio 업로드 시 **'유형'**(`pageType`)으로 올라간다.
3. **`.inner` 의무 폐기** — "모든 콘텐츠를 `.inner` 안에 둔다" 규칙을 폐기한다. 이너 폭은 다양하게
   설정될 수 있고 섹션에 따라 없을 수도 있다.
4. **더 과감한 타이포·레이아웃 + 풀스크린 히어로 기본** — 조금 더 과감한 타이포 배치·레이아웃을 기본으로
   올리고, 메인 히어로는 스크린 풀사이즈 + 오버사이즈 타이포를 기본으로 한다(해외 어워드/국내 에이전시 참고).

## 범위 · 원칙

- 수정 대상 파일: `skills/cova-make-design/SKILL.md`, `.../references/interview.md`,
  `.../references/design-rules.md`, `.../references/self-test.md`,
  `.../references/design-trends.md`, `skills/cova-make-guide/references/design-trends.md`.
- `design-trends.md`는 두 스킬(`cova-make-design`, `cova-make-guide`)에 **byte-identical**로 존재한다.
  변경 1·4는 트렌드 파일을 건드리므로 **두 사본을 동일하게 동기화**한다(diff 0 유지).
- 범위 밖(수정하지 않음): `README.md`, `bin/install.mjs`, `cova-make-guide/references/guide-rules.md`,
  `cova-make-guide/references/templates/*.html`의 `.inner` CSS(HTML 템플릿 내부 클래스로 규칙과 무관).
- 서버·API·payload **필드 스키마 변경 없음**. `pageType` 값(`main`/`subpage`/`dashboard`)은 그대로 유지한다.

## 변경 1 — 트렌드 사용 목적 재정의

파일: `references/design-trends.md` (make-design + make-guide 동일 적용)

- **현재** (상단 도입부):
  > 목적은 하나 — **시안이 "같은 이너폭 안에 갇힌 전형적 구성"으로 수렴하는 것을 막는다.**
- **개정**: 목적을 **"다양한 베리에이션·무드·색감·레이아웃을 차용하기 위한 참고 자료"** 로 재정의한다.
  '이너폭에 갇힌' 프레이밍은 제거한다(변경 3과 정합). 전형성/AI스러움 회피는 목적이 아니라 **부수 결과**로
  한 줄만 남긴다.

## 변경 2 — 페이지 유형 게이트 승격

파일: `references/interview.md`, `SKILL.md`

- `interview.md`에 **라운드 0 — 페이지 유형(게이트)** 를 신설한다. 나머지 인터뷰(라운드 1~4) 앞에 위치.
  - 사용자가 대화에서 이미 유형을 말했으면 **건너뛴다**. 말하지 않았으면 **반드시** AskUserQuestion으로 묻는다.
  - 라벨: **메인 / 서브(하위) / 관리자**. 질문 문구에 "이 선택이 COVA studio 업로드 시 **'유형'**으로
    올라간다"를 명시한다.
  - 매핑(payload 값 유지): 메인 → `main`, 서브 → `subpage`, **관리자 → `dashboard`**.
- 라운드 1의 기존 "페이지 타입" 항목(`main(홈)/subpage/dashboard`)은 **삭제**한다(게이트로 이동, 중복 제거).
- 기존 규칙 "페이지 타입이 `subpage`/`dashboard`면 라운드 2·3은 건너뛴다"는 유지하되, 이제 **게이트 결과**가
  이 분기를 구동함을 문구에 반영한다.
- `SKILL.md` 1단계(요구사항 인터뷰)에: 페이지 유형을 **가장 먼저 게이트로 확정**하고(미언급 시 질문),
  그 값이 6단계 업로드 payload의 `pageType`(= studio '유형')으로 쓰인다는 설명을 추가한다.
- `SKILL.md` 6-2 payload 주석의 `"pageType"(main|dashboard|subpage)` 옆에 "studio 유형" 표기를 덧붙인다.

## 변경 3 — `.inner` 의무 폐기 → 선택적 가변 컨테이너

파일: `references/design-rules.md`, `references/self-test.md`

- `design-rules.md` 레이아웃·정렬 섹션:
  - **현재** (line 5): "기본 콘텐츠는 `.inner`(… `max-width: 1180~1240px` …) 안에 둔다."
    → **개정**: 콘텐츠 폭은 **섹션 의도에 따라 가변**(좁은 읽기 컬럼 / 기본 ≈1180–1240 / 넓게 / 풀블리드)이며,
    `.inner`는 폭을 고정하는 유일한 레일이 아니라 **필요할 때 쓰는 가변 컨테이너**다(없는 섹션도 정상).
  - **현재** (line 6): "풀폭 배경은 `section`이 담당하고 … `.inner`에 배치한다. **의도 없이** 콘텐츠가
    브라우저 끝에 붙으면 실패."
    → **개정**: 판단 기준을 '고정 레일'에서 **'의도된 폭 + 좌측 정렬 기준선'** 으로 전환한다.
    "브라우저 끝에 붙는 게 **의도면 OK**, 의도 없이 붙거나 넘치면 실패." 좌측 정렬 기준선은 유지.
  - line 20 "같은 리듬으로 반복되면(같은 **이너폭**·같은 카드·같은 간격) …" → "같은 **폭**·같은 카드·같은 간격"으로
    표현 정리(이너폭 고정 뉘앙스 제거).
  - line 49 공통 레이아웃 클래스 목록의 `.inner`는 **"가변 폭(옵션)"** 으로 주석한다(삭제하지 않음).
  - line 80 self-check "모든 주요 콘텐츠가 `.inner` 안에 있고, 좌측 정렬 기준선이 …" →
    "각 섹션의 **폭·정렬이 의도적**이고, 좌측 정렬 기준선이 섹션마다 흔들리지 않는가"로 교체.
- `self-test.md` line 14: "콘텐츠가 브라우저 끝까지 퍼지지 않고 `.inner` 안에 있는가." →
  "콘텐츠가 브라우저 끝에 붙는 것이 **의도된 것인가**(의도 없이 끝에 붙거나 넘치지 않는가)."로 교체.

## 변경 4 — 더 과감한 타이포·레이아웃 + 풀스크린 히어로 기본

파일: `references/design-rules.md`, `references/design-trends.md`(양쪽), `references/interview.md`

- `design-rules.md`:
  - 레이아웃/타이포 규칙에 **메인 페이지 히어로 기본값**을 추가: 스크린 풀사이즈(`min-height` ≈ 92–100vh)
    + 오버사이즈 타이포. 타이포 자체를 비주얼로 쓰는 **해외 어워드·국내 에이전시 문법을 적극 참고**하라는 지침.
  - 서브/관리자 유형에는 풀스크린 히어로를 강제하지 않는다(메인 한정 기본).
- `design-trends.md`(양쪽 동일):
  - 무브 #1 "풀블리드 히어로"를 **'이미지 몰입 업종 한정'에서 메인 시안 기본기**로 격상. 이미지형뿐 아니라
    **타이포 중심 풀스크린 히어로**(무브 #10 오버사이즈 헤드라인 · #11 극단적 위계 대비와 세트)를 명시.
  - 히어로 설계 시 해외(Awwwards 등)·국내 에이전시 사이트 사례를 참고하라는 문구를 반영.
- `interview.md` 라운드 4 '디자인 과감함': 기본 추천을 **균형 → 과감**으로 상향(추천 마커 이동).
- **업종 예외 유지**: 병원·금융·공공·B2B 등 신뢰 업종은 풀스크린 히어로·오버사이즈 타이포는 허용하되,
  실험 무브(#4·13·14·17)는 기존 `design-trends.md`의 '업종별 적용 강도' 표대로 절제한다(변경 없음).

## 상호 정합성

- 변경 1(이너폭 프레이밍 제거)과 변경 3(.inner 의무 폐기)은 같은 방향 — 트렌드 목적 문구에서 이너폭 표현을
  빼는 것이 변경 3의 전제가 된다.
- 변경 2(유형 게이트)와 변경 4(메인 한정 풀스크린 히어로)는 연결 — 게이트에서 확정한 유형이 히어로/인터뷰
  분기를 구동한다(메인=풀스크린 히어로 기본 + 라운드 2·3 진행, 서브/관리자=스킵·비강제).
- 변경 4의 기본 상향은 인터뷰 '과감' 기본값 + 규칙의 히어로 기본으로 표현하되, 신뢰 업종의 실험 무브 절제는
  기존 업종별 강도 표가 계속 상한선을 건다(전면 과감 아님).

## 검증

- `diff skills/cova-make-design/references/design-trends.md skills/cova-make-guide/references/design-trends.md`
  가 **빈 출력**(동일)인지 확인한다.
- `.inner` 잔존 참조가 규칙/자체점검 문구에 **의무 뉘앙스로 남지 않았는지** grep으로 확인한다(공통 클래스
  목록의 옵션 표기는 허용).
- 인터뷰 라운드 0(게이트) → 라운드 1~4 순서, 유형 미언급 시 질문/언급 시 스킵 분기가 문서상 일관적인지 통독.
- payload 스키마(`pageType` 값)와 서버는 변경되지 않았음을 확인(문서 표기만 추가).
