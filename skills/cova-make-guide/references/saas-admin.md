# SaaS 어드민 문법 (관리자 시안 · 도구형 제품 · 2026)

관리자 시안과 도구형 제품(Linear·Notion류) 화면의 목표는 하나 — **"실제로 운영 중인 SaaS의
화면"처럼 보이는 것.** 참고 계보: Linear(표면 사다리·크롬 절제), Stripe(데이터 신뢰),
Vercel/Geist(미니멀 정밀), Supabase, Attio.

> **제1원칙. 어드민의 화려함은 장식이 아니라 밀도·정밀함·상태 설계다.**
> 마케팅 페이지의 문법(풀스크린 히어로, 오버사이즈 타이포, 그라데이션, 후기·CTA 섹션)을
> 어드민에 가져오는 순간 "템플릿"으로 읽힌다. 반대로 tabular-nums로 정렬된 숫자, 현실적인
> 목데이터, 절제된 상태색, 일관된 8pt 리듬이 "진짜 제품"을 만든다.
> [design-trends.md](design-trends.md)의 방향 카탈로그·로테이션 규율은 여기 적용하지 않는다 —
> **어드민은 일관성이 곧 전문성**이다.

## 무드 선택 (어드민에도 방향은 있다 — 셋 중 하나)

- **A. 미니멀 정밀 (Linear·Vercel 계보)** — 뉴트럴이 UI의 90%, 액센트 1색, 그림자 대신 헤어라인,
  다크 모드가 잘 어울림. 개발자 도구·인프라·데이터 제품. **기본값.**
- **B. 데이터 신뢰 (Stripe 계보)** — 라이트 고정, 화이트+블루그레이(#f6f8fa류) 배경 2단,
  인디고/블루 계열 액센트, 표·수치 중심. 금융·커머스·정산·CRM.
- **C. 프로덕트 친화 (토스·국내 서비스 계보)** — 조금 더 둥글고(radius 8~10) 부드러운 뉴트럴,
  선명한 브랜드 1색. 단 밀도와 정밀함은 A/B와 동일하게 유지. 교육·커뮤니티·사내 도구.

인터뷰의 참고 무드 답으로 고르고, 답이 없으면 업종으로 판단(기술적일수록 A, 금전적일수록 B).

## 앱 셸 규격 (수치)

- **사이드바**: 폭 240~280px(기본 256px). 구조는 위에서부터 — 워크스페이스/로고 영역(높이 48~56px),
  (선택) 검색 트리거, 내비 그룹들, 하단 고정 유저 카드. 배경은 본문과 명도 1단 분리해 사이드바가
  물러나게 한다(라이트: 본문 #ffffff에 사이드바 #f7f8f9 / 다크: 사이드바 #101012에 본문 #161719 —
  아래 '다크 모드'의 표면 사다리와 같은 값이다. Linear 방식은 사이드바를 의도적으로 '더 흐리게'
  해서 본문에 시선을 몰아준다).
- **내비 아이템**: 높이 32~36px, 좌우 패딩 8~12px, radius 6px, 아이콘 16px + 라벨 13~14px,
  hover 배경은 라이트 `rgba(0,0,0,.04~.06)` / 다크 `rgba(255,255,255,.04~.06)`,
  active는 액센트 8~10% 배경 + 글자색은 잉크 최상단(#171717 / 다크 #f7f8f8)이 기본(무드 C만
  액센트색 텍스트 허용). 그룹 라벨은 11~12px, 보조색, 위 마진 20~24px — 영문 라벨이면
  uppercase + `letter-spacing: .04~.06em`, 국문 라벨은 크기·색으로만 구분.
- **상단바**: 높이 48~56px. 좌측 브레드크럼(13px, 구분자 `/`), 중앙 또는 우측에 검색 인풋
  (placeholder "검색" + 우측 끝 `⌘K` 키캡 표시 — 시안에서는 형태만), 우측에 아이콘 버튼 1~2개 + 아바타.
  하단 1px 헤어라인으로 본문과 구분.
- **콘텐츠 영역**: 패딩 24~32px. 최대 폭 기준 — 테이블 중심 화면은 풀-플루이드,
  대시보드·폼·설정은 1200~1440px. 페이지 헤더 = 타이틀(16~20px/600) + 우측 primary 액션 버튼
  1개(초록·파랑 큰 버튼 남발 금지 — 화면당 primary 1개).
- **뷰포트**: 어드민 시안은 **데스크톱 기준(1280~1440px)으로 설계·검사**한다. 1024px 미만은
  사이드바를 아이콘 레일(56~64px)로 축소하는 형태만 허용하고, 시안 한 장에서 모바일 최적화는
  하지 않는다(테이블은 컨테이너 `overflow-x: auto` 스크롤). 390px 1열 검사는 어드민에 적용하지 않는다.

## 타이포·밀도

- **본문 13~14px / 행간 1.5** (마케팅 페이지의 16~18px과 완전히 다르다 — 이 차이가 어드민답게
  만드는 절반이다). 보조 라벨 12px, 그룹 헤더 11~12px uppercase, 페이지 타이틀 16~20px semibold,
  KPI 숫자 24~32px. 자간: 소형 텍스트 0, 타이틀 -0.01em.
- **폰트: Pretendard 단독**(어드민은 디스플레이 폰트 로테이션 대상이 아니다) + 숫자·ID·코드에
  모노(`ui-monospace` 또는 JetBrains Mono) 보조.
- **간격: 4px 서브그리드의 8pt 스케일** — 4/8/12/16/24/32/40. "그룹 내부 8px, 그룹 간 16px,
  섹션 간 24~32px"의 리듬. 임의값(13px, 37px) 금지.
- **tabular-nums**: 세로로 쌓이는 모든 숫자(테이블 컬럼·금액·통계)에
  `font-variant-numeric: tabular-nums` 적용. 단 큰 단독 숫자(KPI 값)는 기본 비례 숫자가 예쁘다 —
  컬럼에만 tabular.

## 색 체계

- **뉴트럴 사다리가 UI의 90%.** 라이트: 배경 2단(#ffffff / #fafafa), 잉크 #171717(순검정 금지),
  보조 텍스트 #4d4d4d~#707070, 뮤트 #9a9a9a(placeholder·비활성 전용 — 읽어야 하는 보조 라벨·축
  라벨은 #707070 이상), 보더 #e5e5e5/#ededed(2단).
- **브랜드 액센트 1색** — primary 버튼·포커스 링·활성 상태·링크에만. 계보 참고: Linear #5e6ad2,
  Stripe #635bff, Supabase #3ecf8e, Attio #3abdaf. 액센트를 카드 배경·아이콘 타일·구분선에
  퍼뜨리지 않는다(Supabase의 명문 규칙: "그린은 버튼 전용, 카드에는 쓰지 않는다").
- **시맨틱 색은 예약어** — success/warning/danger/info는 상태 뱃지·알럿·검증 문구에만.
  뱃지 공식: 같은 색조의 **저채도 배경(10~15% 틴트) + 진한 텍스트** (예: `background: #e7f6ec;
  color: #116329;`). 차트 시리즈 색으로 시맨틱 색 재사용 금지.
- **다크 모드(테마 답이 '다크'일 때)** — 표면 사다리 한 벌로 통일한다: 사이드바·페이지 바탕
  #101012 → 본문 배경 #161719 → 카드/표면 #1b1c1f → hover·active #202126. 텍스트
  #f7f8f8 / #d0d6e0 / #8a8f98 3단(순백 금지), 헤어라인 #26262b, 차트 그리드 #26262b.
  깊이는 그림자가 아니라 이 명도 단계로만 — 다크에서 그림자·컬러 glow는 0. hover는
  `rgba(255,255,255,.04~.06)`. 시맨틱 뱃지·알럿은 라이트의 틴트 공식 대신 **알파 공식**:
  같은 색조 12~16% 알파 배경 + 밝은 텍스트(예: `background: rgba(62,207,142,.14); color: #4ade80;`).
- **그림자는 억제** — 카드 그림자는 없거나 `0 1px 2px rgba(0,0,0,.04)` 수준(불투명도 0.02~0.06).
  구분은 1px 헤어라인 보더가 담당. 모달/팝오버만 `0 8px 16px -4px rgba(0,0,0,.08)` 정도.
  헤어라인 보더와 넓게 퍼진 그림자를 동시에 쓰면 AI 티가 난다.
- **radius**: 버튼·인풋·카드 6~8px, 메뉴·모달 10~12px가 상한, pill(9999px)은 뱃지·아바타만.
  12px 초과 금지(soft blob 카드 = 안티패턴). 무드 C만 카드 8~10px 허용.
- **포커스 링**: `box-shadow: 0 0 0 2px <요소가 놓인 실제 배경색>, 0 0 0 4px var(--color-primary);`
  — 갭 색은 bg가 아니라 요소가 놓인 배경(bg-soft 위 버튼이면 bg-soft)을 따른다. 인풋·버튼에 일관 적용.

## 컴포넌트 문법 (스펙)

- **데이터 테이블** — 어드민의 주인공. 헤더: 12~13px/500~600 보조색, `position: sticky; top: 0`,
  배경 불투명 지정. 행: 높이 40~48px(밀집 36~40px), 셀 좌우 패딩 12~16px, 하단 1px 헤어라인,
  hover 배경 1단. 숫자 컬럼 우측 정렬 + tabular-nums, 텍스트 좌측, 상태 중앙. 첫 열 체크박스,
  끝 열 kebab(⋯) 액션. 상태 표시 = 8px dot + 라벨 텍스트(또는 저채도 뱃지) — 색만으로 상태를
  전하지 않는다. 테이블 하단: "총 1,284건 중 1–20" + 페이지네이션(‹ 1 2 3 … ›, 현재 페이지만 액센트).
- **스탯 카드(KPI)** — 계약: 라벨(12~13px 보조색, 문장형·콜론 없음) / 값(24~32px semibold,
  긴 수는 12.4K·₩3.2M식 축약) / 델타(±부호 + 비교 기간 명시 "지난주 대비 +4.2%", 색은
  방향×좋음 — 이탈률 감소는 초록) / (선택) 12포인트 스파크라인. 스트립은 카드 4~6개
  `repeat(auto-fit, minmax(200px, 1fr))`. **"큰 숫자 + 그라데이션 라인 + 작은 라벨" 히어로
  메트릭 클리셰 금지** — AI 대시보드의 90%가 이 모양이다.
- **필터 툴바** — 테이블 위 한 줄: 검색 인풋(높이 32~36px) + 필터 칩/셀렉트 2~3개 +
  우측 끝 primary 액션. 적용된 필터는 X 있는 칩으로 표시.
- **폼** — 라벨 상단 배치 12~13px/500, 인풋 높이 32~36px + 패딩 8~12px + 1px 보더 + radius 6px,
  placeholder는 뮤트색, 도움말/검증 문구는 인풋 아래 12px(danger). 필드 간 16px, 그룹 간 24px.
  버튼: primary(액센트 채움)/secondary(보더)/ghost(투명) 높이 32~36px, 라벨 13~14px/500~600.
  primary hover는 액센트 명도 -6~8%(= `--color-primary-strong`), disabled는 `opacity: .5` +
  `cursor: not-allowed`.
- **모달/드로어** — 모달 폭 400~560px, radius 10~12px, 헤더(타이틀+X)/본문/푸터(우측 정렬 버튼).
  사이드 드로어는 폭 380~480px. 배경 오버레이 `rgba(0,0,0,.4)`.
- **탭** — 텍스트 13~14px, 활성 = 하단 2px 액센트 인디케이터 + 글자 진하게. 박스형 탭 금지.
- **알럿/토스트** — 알럿: 시맨틱 배경 8~12% 틴트 + 1px 보더 + 아이콘 16px + 13px 텍스트.
  토스트: 다크 표면(#1c1d1f) + 흰 텍스트, 우하단.
- **엠티 스테이트** — 아이콘 또는 절제된 일러스트 1 + 상황 설명 한 문장 + 행동 CTA 1개
  ("아직 등록된 상품이 없습니다" + [상품 등록]). 빈 화면은 행동의 초대장.
- **스켈레톤** — 실제 콘텐츠 모양 그대로(테이블이면 행 모양, 카드면 카드 모양)의 회색 블록.
  시안에서는 한 영역 정도에 로딩 상태를 보여주면 "실제 제품" 인상이 강해진다.
- **아바타** — 이니셜 2자 + 뉴트럴 배경(이름별로 8~10색 뮤트 팔레트 순환), 크기 24~32px, 원형.
- **아이콘** — 이모지 절대 금지. 인라인 SVG 스트로크 아이콘(16~20px, stroke-width 1.5~1.75,
  `currentColor`)로 통일. 필요한 아이콘(홈·문서·사용자·설정·검색·벨·플러스·케밥·화살표)은 직접 그린다.

## 진짜처럼 보이는 목데이터 (전문성의 절반)

- **이름**: 한국 서비스면 현실적 한국 이름 분포(김서연, 이준호, 박민지, 최현우, 정다은…) —
  "홍길동·김철수" 반복은 즉시 가짜. 글로벌 톤이면 문화권 혼합.
- **이메일·ID**: 이름에서 파생(`seoyeon.kim@...`, `jh.lee@...`). ID는 실제 서비스 형식
  (`INV-2026-0142`, `ord_8f3kx2`, `#4821`)으로 연속성 있게.
- **금액**: 우수리 있는 계산된 값(₩1,247,500 / ₩89,000 / ₩12,340,000) — ₩1,000,000 같은
  라운드 넘버 나열 금지.
- **날짜**: 최근에 밀집(오늘·어제·이번 주 다수, 과거로 갈수록 희소). "방금 전 / 2시간 전 /
  어제 / 2026-07-08" 상대·절대 혼용, 형식은 한 테이블 안에서 일관.
- **상태 분포**: 현실 비율 — 결제면 완료 85~95%, 대기 소수, 실패/환불 극소수. 상태가 1/3씩
  균등하면 즉시 가짜로 보인다.
- **빈 값**: 일부 셀은 비어 있거나 `—` 처리 — 완벽히 채워진 표가 오히려 부자연스럽다.
- **차트 데이터**: 매끈한 우상향 금지 — 주말 하락, 스파이크, 정체 구간이 있는 현실적 곡선.
- **한 화면의 숫자는 하나의 이야기다** — KPI 델타 방향 = 차트 최근 구간의 방향 = 테이블 최신
  행들의 경향이 서로 맞물려야 한다. 총계·건수도 표시된 데이터와 자릿수 수준에서 정합하게.
  ("+4.2%" 옆에 급락 차트가 있으면 실사용 SaaS 인상이 즉시 깨진다.)
- **리스트 화면은 12~20행** — 3~4행짜리 테이블은 즉시 가짜로 보인다.
- 이 문서와 템플릿의 이름·금액 예시는 분포 감각의 참고일 뿐 — **그대로 복사하지 말고 매번
  새로 생성한다**(모든 시안의 첫 행이 같은 이름이면 그게 새로운 홍길동이다).

## 대시보드·차트 (CSS-only)

- KPI 스트립(4~6개) → 차트 1~2개 → 최근 활동 테이블의 3단 구성이 기본. 뷰당 코어 요소 5~9개,
  가장 큰 "히어로 숫자"는 1개만.
- 차트는 CSS로 그린다: 바 차트(flex 컬럼 + 높이 %), 스파크라인(SVG polyline 인라인),
  도넛(conic-gradient), 프로그레스(선형). 마크는 얇게 — 라인 2px, 바 폭 ≤24px + 끝 4px 라운드,
  면 채움은 10% 불투명 워시. 그리드는 1px 실선 연회색(점선 금지), 축 라벨 11~12px 뮤트.
- **이중축 금지.** 시리즈 2개 이상이면 범례 필수. 값 라벨은 끝점·극값에만(전 포인트 라벨 금지).
  텍스트에 데이터 색을 입히지 않는다(색은 마크가, 텍스트는 잉크가).

## 모션

- 120~200ms `ease-out`의 hover/펼침 트랜지션만. bounce/elastic easing 금지, 스크롤 리빌·패럴랙스
  금지(어드민은 즉시성이 미덕), 장식 모션 0. 스켈레톤 shimmer 정도가 허용되는 최대치.

## 안티패턴 ("AI가 만든 어드민" 신호)

- 보라-인디고 그라데이션(버튼·배경·텍스트 어디든), 다크 배경 + 컬러 glow box-shadow.
- **카드 좌측 3~4px 컬러 보더** — 단일 최강 인식 신호.
- radius 12px 초과(16~24px)의 soft blob 카드, 카드 안 카드 안 카드(패딩+그림자 중첩).
- 이모지 아이콘, 헤딩 위 라운드 아이콘 타일 3개 반복.
- "큰 숫자+작은 라벨+그라데이션 액센트" 히어로 메트릭, 의미 없는 상태 dot.
- 행 높이 52px+(모바일 터치 기준을 데스크톱에 적용한 것), 과한 여백의 낮은 밀도.
- 균등한 상태 분포·홍길동 목데이터·라운드 넘버 금액.
- 마케팅 요소 혼입: 풀스크린 히어로, 후기 섹션, CTA 배너, 오버사이즈 타이포.
- 전부 같은 간격(리듬 없음), 플랫한 폰트 위계(13/14/16의 구분 없는 사용).

## 어드민 시안 페이지 골격

인터뷰의 핵심 화면 답에 따라 셸(사이드바+상단바)은 공통으로 깔고 본문을 구성한다:

- **대시보드 홈**: 페이지 헤더(+기간 필터) → KPI 스트립 → 차트 1~2 → 최근 활동 테이블.
- **리스트 화면**: 페이지 헤더(+primary 액션) → 필터 툴바 → 테이블 → 페이지네이션.
- **상세 화면**: 브레드크럼 → 헤더(이름+상태 뱃지+액션) → 2컬럼(좌 정보 카드들 / 우 활동 로그).
- **폼/설정 화면**: 좌측 섹션 내비(선택) + 폼 그룹들 + 하단 고정 저장 바.

한 장 시안이면 "대시보드 홈" 또는 "리스트"가 대표성이 가장 좋다. 사이드바 내비 항목은
그 서비스의 실제 메뉴 구조(6~9개 + 그룹 라벨)로 채운다 — "메뉴1/메뉴2" 금지.
