# 디자인 시스템 인터뷰

디자인 시스템(토큰·컴포넌트)을 확정하기 전에 아래를 사용자에게 묻는다. **모든 질문은 AskUserQuestion
도구(카드 UI)로** 묻는다. 굵게 표시한 항목은 필수. 이미 대화에서 답한 항목은 다시 묻지 않는다.

## 카드 질문 규칙

- 한 호출에 질문 최대 4개까지 묶는다 — 아래 **라운드 단위로** 나눠 묻는다.
- 선택지형은 옵션 2~4개에 짧은 설명을 붙인다. 자유 입력형은 대표 예시 2~3개 + Other(직접 입력).
- 복수 선택이 자연스러운 질문(예: 필요 컴포넌트)은 multiSelect를 켠다.

## 라운드 1 — 기본

| 질문 | 헤더 | 옵션 구성 |
|---|---|---|
| **스코프** — 이 디자인 시스템을 어디에 쓸지 | 스코프 | 사용자페이지(main/sub) / 제품·앱(user 산출물 — 실제 제품 화면 참고) / 관리자 시스템(admin) |
| 서비스·브랜드 성격 | 성격 | 신뢰·전문(병원·금융·B2B) / 활동·젊음 / 따뜻·친근(로컬·교육) / 공공·안내형 (그 외 Other) |
| **핵심 톤&무드** | 톤 | 절제된 고급 / 선명·경쾌 / 부드러움·라운드 / 도구적·고밀도(관리자 향) |

## 라운드 2 — 방향·색

스코프에 따라 첫 질문이 갈린다. **방향/무드 답이 타이포·라운드·섀도우의 기본값을 결정**하고,
나머지 질문은 그 위에 얹는 조정값이다.

| 질문 | 헤더 | 옵션 구성 |
|---|---|---|
| (user/제품이면) **디자인 방향** — design-trends.md 카탈로그로 매핑 | 방향 | 알아서 제안(추천 — 브리프에서 도출해 커밋, 후보가 팽팽하면 2안 카드로 재질문) / 세련·고급 계열(D1 에디토리얼·D2 콰이어트 럭셔리·D11 모던 한국 명조) / 임팩트·화려 계열(D3 다크 시네마틱·D6 도파민 팝·D7 오로라 글로우) / 친근·활기 계열(D12 소프트 프로덕트·D8 콜라주 스크랩북·D10 오가닉 내추럴) — 구체 방향명은 Other |
| (admin이면) **참고 무드** — saas-admin.md 무드로 매핑 | 무드 | 미니멀 정밀(Linear·Vercel류 — 추천) / 데이터 신뢰(Stripe류) / 프로덕트 친화(토스류) |
| 주색(primary) 선호 | 주색 | 없음(방향에 맞게 알아서) / 블루 계열 / 그린 계열 / 뉴트럴+포인트 (브랜드 컬러는 Other로 hex 입력) |
| 중립 배경 성향 | 배경 | 밝은 화이트 기반 / 오프화이트·소프트 / 다크 UI |
| 모서리 라운드 | 라운드 | 방향 기본값(추천) / 각짐(0~4px) / 표준(6~10px) / 둥긂(12~20px) |

## 라운드 3 — 컴포넌트 범위

| 질문 | 헤더 | 옵션 구성 |
|---|---|---|
| 필요 컴포넌트(복수) — 스코프 기본 셋에서 가감 | 컴포넌트 | (user 기본) 버튼·카드·폼·뱃지·내비·CTA / (admin 기본) 버튼·인풋·테이블·폼·모달·탭·사이드바·스탯·알럿·필터툴바·엠티스테이트 — multiSelect |
| 상태 표현 범위 | 상태 | 기본만 / 기본+hover / 기본+hover+disabled(+focus) — 추천 |

## 라운드 4 — 마무리

| 질문 | 헤더 | 옵션 구성 |
|---|---|---|
| **디자인 과감함** — 커밋한 방향의 볼륨(user 전용, admin은 묻지 않음) | 과감함 | 절제(방향 유지 + 조용한 시그니처 — 절제 ≠ 무난) / 균형 — 추천 / 과감(실험적) |
| **추가 요청사항** — 금지 색·참고 사이트·특정 컴포넌트 등 | 요청사항 | "없음" + Other 직접 입력 |
