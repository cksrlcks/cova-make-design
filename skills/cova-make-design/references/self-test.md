# 자체 테스트 (생성 직후, 반드시)

만든 HTML을 실제로 열어 렌더링·정렬·폰트를 점검한다. Playwright MCP가 있으면 그걸로, 없으면
`npx playwright` 또는 로컬 정적 서버로 연 뒤 스크린샷/DOM을 확인한다:

```bash
# 예: 데스크톱 + 모바일 폭에서 스크린샷
npx --yes playwright screenshot --viewport-size=1440,2200 design-<slug>.html shot-desktop.png
npx --yes playwright screenshot --viewport-size=390,1600  design-<slug>.html shot-mobile.png
```

## 1. 렌더링·정렬 (문제가 있으면 고치고 다시 확인)

- 레이아웃이 깨지거나 요소가 겹치지 않는가. 콘텐츠가 브라우저 끝에 붙는 것이 의도된 것인가.
- 섹션들의 좌측 정렬 기준선이 어긋나지 않는가.
- **뱃지·버튼·칩의 텍스트가 세로 정중앙에 오는가** (design-rules.md의 flex 규칙 — 가장 흔한 결함).
- 선언한 폰트가 **전부** 실제 적용됐는가 — 본문(Pretendard 등)과 헤드라인 디스플레이 폰트 각각
  확인(시스템 폰트로 폴백되지 않았는가). `document.fonts.check('700 1em <폰트명>')`로 검사 가능.
- 사용한 외부 이미지가 실제로 로드되는가(`img.naturalWidth > 0`, 깨진 이미지는 교체).
- 모바일 폭(≈390px)에서 1열로 자연스럽게 정리되는가. 오버랩·회전 요소가 모바일에서 풀리는가.
  **예외: 관리자·도구형 제품은 390px 검사 대신 1024px에서 사이드바 축소(아이콘 레일)와 테이블
  overflow 스크롤이 동작하는지 확인**(saas-admin.md 뷰포트 규칙).
- 의도치 않은 가로 스크롤이 없는가(편측 흘림을 썼다면 `html { overflow-x: clip }` 확인).

## 2. 개성·클리셰 스캔 (사용자 페이지)

- **HTML 최상단에 design-plan 주석이 존재하는가**(design-rules.md §0) — 없으면 1차 패스를
  건너뛴 것이므로 실패.
- **주석의 선언과 실제 CSS가 일치하는가** — 주석의 팔레트 hex·폰트명·무브 번호를 grep으로
  본문 CSS와 대조한다(플랜 따로 결과 따로 금지). D2·D4 커밋이면 "비튼 축" 2개가 실제 구현됐는가.
- **시그니처 요소가 실제로 존재하는가** — 스크린샷에서 "이 시안만의 한 방"이 보이는가.
- AI 클리셰가 없는가: 보라-인디고 그라데이션 / 아이콘 3열 카드 / 카드 좌측 컬러 보더 /
  이모지 아이콘 / pill eyebrow + 중앙 히어로 템플릿 / 의미 없는 01·02 번호 / "혁신적인" 류 카피 /
  AI 디폴트 룩 3종 공식 그대로.
- 섹션 레이아웃이 4종 이상으로 변주되는가(같은 카드 그리드 반복 아님).

## 3. 어드민 추가 점검 (관리자·도구형 제품)

- 본문이 13~14px 밀도인가(마케팅 사이즈 16~18px로 커지지 않았는가). 행 높이 40~48px(밀집 36~40).
- 테이블 숫자가 **우측 정렬 + tabular-nums**인가. 헤더가 sticky인가.
- 상태 표시가 dot/뱃지 + 라벨 텍스트인가(색만으로 전달 금지), 뱃지가 저채도 배경+진한 텍스트 공식인가.
- 목데이터가 현실적인가 — 홍길동 반복·라운드 넘버 금액·균등 상태 분포·전부 같은 날짜가 없는가.
- 마케팅 요소(히어로·후기·CTA 배너·오버사이즈 타이포)가 섞여 들어오지 않았는가.
- 그림자가 억제되어 있는가(헤어라인 보더 위주), radius ≤ 12px, 이모지 아이콘 0개.

## 4. 모션·폴백

- 애니메이션 대상의 **기본 상태가 완성형**인가 — `opacity: 0`이 `@supports`/`@media` 게이트 밖에
  있으면 미지원 브라우저에서 콘텐츠가 사라진다(치명 결함).
- `prefers-reduced-motion`을 존중하는가. transform/opacity 외 속성을 애니메이션하지 않는가.
- scroll-driven·backdrop-filter 등은 미지원 환경에서 자연스럽게 열화되는가.

## 팁

- 스크린샷을 눈으로 확인할 수 없으면 DOM 검사로 대신한다:
  뱃지/버튼 요소의 `getBoundingClientRect()`와 내부 텍스트 노드의 rect를 비교해
  위/아래 여백 차가 2px 이내인지 확인하는 식.
- 이미지 로드는 `page.on('response')` 또는 각 `img`의 `naturalWidth > 0`으로 확인할 수 있다.
- 클리셰 스캔은 grep으로도 일부 가능: `grep -c "linear-gradient(.*#[68].*f" ` (보라 계열),
  이모지 유니코드 범위, `border-left: [34]px` 등.
