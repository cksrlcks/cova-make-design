# 요구사항 인터뷰

HTML을 만들기 전에 아래를 사용자에게 묻는다. **모든 질문은 AskUserQuestion 도구(카드 UI)로
묻는다** — 일반 텍스트로 질문을 나열하지 않는다. 굵게 표시한 항목은 필수다.
(카드 UI가 없는 환경에서만 같은 라운드 구성 그대로 텍스트로 묻는다.)

## 카드 질문 규칙

- AskUserQuestion 한 호출에 질문을 최대 4개까지 묶을 수 있다 — 아래 **라운드 단위로** 나눠 묻는다.
- 선택지형 질문은 옵션 2~4개에 각각 짧은 설명을 붙인다.
- 자유 입력형 질문(제목·슬로건 등)도 카드로 묻는다: 대표 예시를 옵션으로 2~3개 제시하면
  사용자가 "Other(직접 입력)"로 원하는 값을 넣을 수 있다. 질문 문구에 직접 입력을 안내한다.
- 복수 선택이 자연스러운 질문은 multiSelect를 켠다.
- 사용자가 이미 대화에서 답한 항목은 다시 묻지 않는다.
- 페이지 유형(라운드 0)이 `subpage`면 라운드 2·3에서 필요한 것만 골라 묻는다.
  `dashboard`면 라운드 2·3 대신 **라운드 A**, `product`면 **라운드 P**를 묻는다.

## 라운드 0 — 페이지 유형 (게이트, 가장 먼저)

시안을 시작할 때 **어떤 페이지를 만들지 먼저 확정한다.** 사용자가 대화에서 이미 유형을 말했으면
이 라운드를 건너뛰고, 말하지 않았으면 **반드시** 묻는다. 이 선택은 COVA studio 업로드 시
**'유형'**(payload `pageType`)으로 올라간다.

| 질문 | 헤더 | 옵션 구성 |
|---|---|---|
| **페이지 유형** — studio '유형'으로 올라간다 | 유형 | 메인(`main`) / 서브·하위(`subpage`) / 관리자(`dashboard`) / 제품(`product`) |

- **메인** → 라운드 1~4 진행. [design-trends.md](design-trends.md)의 방향 커밋 게이트를 따른다.
- **서브** → 라운드 2·3은 필요한 것만. 방향은 메인과 같은 논리로 커밋(볼륨은 한 단계 낮게).
- **관리자** → 라운드 2·3 대신 **라운드 A**. 규칙은 [saas-admin.md](saas-admin.md).
- **제품**(바이브코딩 제품·앱) → 라운드 2·3 대신 **라운드 P**. 도구형(어드민류)이면
  [saas-admin.md](saas-admin.md), 소비자형이면 design-trends.md의 D12(소프트 프로덕트) 축을 참고.

## 라운드 1 — 기본 (모든 시안)

| 질문 | 헤더 | 옵션 구성 |
|---|---|---|
| **사이트 제목** — 업로드 시 title로 쓰인다. 직접 입력 안내 | 제목 | 예시 옵션: "은성한의원 메인" / "무브핏 랜딩" + Other 직접 입력 |
| 업종/서비스 | 업종 | 병원·의원 / 학원·교육 / 카페·로컬 매장 / IT·SaaS (그 외는 Other) |
| 페이지 목적 | 목적 | 신뢰·상담 유도 / 예약·문의 / 제품·구매 / 브랜드 소개 (캠페인은 Other) |

## 라운드 2 — 메인·서브(main/subpage)면: 방향 (서브는 이 중 필요한 질문만)

| 질문 | 헤더 | 옵션 구성 |
|---|---|---|
| **디자인 방향(무드)** — design-trends.md 방향 카탈로그로 매핑된다 | 방향 | 알아서 제안(추천 — 브리프에서 도출해 커밋, 후보가 팽팽하면 2안 카드로 재질문) / 세련·고급 계열(D1 에디토리얼·D2 콰이어트 럭셔리·D11 모던 한국 명조) / 임팩트·화려 계열(D3 다크 시네마틱·D6 도파민 팝·D7 오로라 글로우) / 친근·활기 계열(D12 소프트 프로덕트·D8 콜라주 스크랩북·D10 오가닉 내추럴) — 구체 방향명·참고 사이트는 Other |
| 대표 CTA 하나 | CTA | 문의 / 예약 / 구매 / 가입 (다운로드 등은 Other) |
| 섹션 구성 | 구성 | 업종에 맞게 알아서 / 전환 중심(가격·FAQ·CTA 강화) / 스토리 중심(소개·사례·후기 강화) / 정보 중심(절차·안내·연락 강화) — 특정 섹션 지정은 Other |
| 이미지 톤 | 이미지 | 실사(무료 이미지로 채움) / 제품컷 / 일러스트 |

## 라운드 3 — 메인·서브(main/subpage)면: 내용 (서브는 이 중 필요한 질문만)

| 질문 | 헤더 | 옵션 구성 |
|---|---|---|
| 한 줄 소개(슬로건)/핵심 가치 — hero 카피의 뼈대 | 슬로건 | "알아서 제안" 옵션 + Other 직접 입력 |
| 타깃 고객 | 타깃 | 지역 주민·방문객 / 20~30대 / 기업·기관 담당자 / 전연령 (그 외 Other) |
| 브랜드 컬러·선호 색감 | 색감 | 없음(방향에 맞게 알아서) / 차분한 저채도 / 선명한 포인트 컬러 (브랜드 컬러는 Other로 hex 입력) |
| **브랜드 고유 재료·실제 데이터** — 지역, 창업 스토리, 대표 메뉴/시술, 공간 특징, 차별점, 상호명·연락처·후기 등. **방향 커밋(게이트 ②)의 원료**가 된다 | 고유 재료 | 있음(Other로 구체적으로 전달 — 많을수록 시안이 고유해진다) / 없음 — 업종 관행 기반 + 자연스러운 예시로 채움(지어내지 않기) |

## 라운드 A — 관리자(dashboard)면: 실사용 SaaS처럼 (라운드 2·3 대신)

유형이 **관리자**일 때만 묻는다. 답은 [saas-admin.md](saas-admin.md)의 무드·골격 선택에 쓴다.

| 질문 | 헤더 | 옵션 구성 |
|---|---|---|
| 참고 무드 — 어떤 SaaS 계보를 닮게? | 무드 | 미니멀 정밀(Linear·Vercel류 — 추천) / 데이터 신뢰(Stripe류, 표·수치 중심) / 프로덕트 친화(토스류, 부드럽지만 밀도 유지) |
| **대표 화면** — 이번 시안 한 장으로 그릴 화면 | 화면 | 대시보드 홈(KPI+차트+최근 활동) / 리스트·테이블(필터+테이블+페이지네이션) / 상세 화면(정보+활동 로그) / 폼·설정 |
| 관리 대상 — 테이블·메뉴를 채울 실제 도메인 | 도메인 | 주문·결제 / 회원·고객 / 콘텐츠·게시물 / 예약·일정 (그 외 Other — 구체적으로) |
| 테마·밀도 | 테마 | 라이트 표준 밀도(추천) / 라이트 고밀도(행 36~40px) / 다크 |

## 라운드 P — 제품이면: 실제 제품 화면 참고 (라운드 2·3 대신)

유형이 **제품**일 때만 묻는다. 마케팅 사이트 문법(히어로·후기·CTA) 대신 **실제 제품/앱 UI 관례**를
참고한다. 도구형이면 [saas-admin.md](saas-admin.md)의 셸·밀도 규격을 그대로 쓴다.

| 질문 | 헤더 | 옵션 구성 |
|---|---|---|
| 참고 제품·레퍼런스 — 어떤 실제 제품/앱을 닮게? | 레퍼런스 | 도구형(Linear·Notion류 → saas-admin.md 적용) / 서비스앱(Toss·배민류) / 어드민·대시보드 툴(→ saas-admin.md 적용) (구체 사이트는 Other 직접 입력) |
| 핵심 화면 — 이번에 그릴 제품 화면(복수) | 화면 | 리스트·피드 / 상세·디테일 / 폼·입력 흐름 / 대시보드·홈 — multiSelect (그 외 Other) |
| 주요 UI 패턴 — 필요한 구조(복수) | 패턴 | 상단바+콘텐츠 / 사이드바+콘텐츠 / 리스트+디테일 2분할 / 카드 그리드 — multiSelect |
| 정보 밀도·톤 | 밀도 | 도구형 고밀도(정보 촘촘) / 균형 / 소비자형 여백감 |

## 라운드 4 — 마무리 (모든 시안, 필수)

| 질문 | 헤더 | 옵션 구성 |
|---|---|---|
| **디자인 과감함** — 커밋한 방향의 볼륨(개수가 아니라 강도) | 과감함 | 절제(방향은 유지하되 조용한 시그니처 1개 — 절제 ≠ 무난) / 균형(시그니처 1 + 무브 3~4 + 모션 1) — 추천 / 과감(실험적 레이아웃 + 모션 2~3, 방향 최대 볼륨) |
| **추가 요청사항** — 특정 섹션, 금지 요소, 카피 톤, 참고 사이트 등 | 요청사항 | "없음" 옵션 + Other 직접 입력 |
| **완성되면 COVA에 업로드할까요?** — 답을 받아 6단계 진행 여부 결정 | 업로드 | 예 — 완성 후 업로드 / 아니오 — 로컬 파일만 |
| 태그 선택 흐름 | 태그 | 알아서 — 인터뷰 답을 태그로 매핑 / 섹션별 — 히어로·서비스·후기·CTA 직접 선택 |

- 관리자(dashboard) 유형의 과감함은 "밀도·디테일의 완성도"로 해석한다 — saas-admin.md의
  문법을 벗어나는 장식적 실험은 과감을 골라도 하지 않는다.
